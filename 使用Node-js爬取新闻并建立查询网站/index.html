使用Node.js爬取新闻并建立查询网站

发表于 2021-04-23 更新于 2021-06-07

一、需求

  • 选取3-5个有代表性的新闻网站建立爬虫，针对不同网站的新闻页面进行分析，爬取出标题、链接、摘要、关键词、时间、来源、责编等结构化信息，存储至数据库中
  • 建立网站提供对爬取内容进行搜索，给出所查询关键词的时间热度分析
  • 技术要求
    • 采用Node.js实现网络爬虫
    • 采用Node.js实现查询网站后端，HTML+JavaScript实现前端
    • 尽量不使用任何前后端框架

3.1 新闻条目爬取

由于我们需要爬取该网站下的多个子项目的新闻列表，因此首先创建一个URL列表存储该网站下所有子项目的链接：

然后我们需要遍历该列表，爬取每一个URL下的新闻信息。由于Node.js的异步特性，使用for循环遍历并爬取数据时可能会由于突然的大量并发而导致网络阻塞从而造成数据加载失败，因此我们需要引入 async 进行并发控制。

使用 async.mapLimit 进行并发控制，第一个参数为一个可迭代对象，第二个参数为并发数量，这里我们设置为1，每次仅爬取一个URL，第三个参数为所要执行的异步函数，其参数为 urls 中的每一个数据成员。

接下来我们需要初始化一个浏览器并打开我们想要爬取的页面：

  • options 用来设置浏览器的一些参数，headless:false 表示打开一个浏览器界面（默认状态下是无界面的）,defaultViewport 用来设置打开的浏览器的窗口大小
  • puppeteer.launch(options) 启动一个浏览器
  • browser.newPage() 打开一个新的页面
  • page.setDefaultNavigationTimeout(0) 设置浏览器的加载时长不限，避免因网络原因加载缓慢而直接返回错误
  • page.setRequestInterception(true) 设置拦截图片加载，避免因加载图片占用网络带宽造成延迟
  • page.gotu(url) 页面跳转至指定的 URL

七、总结

关于该项目，实现了：

  • 数据爬取 & 存储

  • 倒排索引构建 & 存储

  • 多查询词查询

  • 查询词时间热度分析与可视化

  • 前端设计与实现

在此之前也写过爬虫，写过后端，写过前端，但是这是第一次使用 Node.js 写项目，而 Node.js 与之前所使用的编程语言有很明显的不同，最大的不同可能就是 Node.js 是基于异步的编程语言。因此，在开始之初的确很不适应这种编程风格（PS：即使现在也还是不适应），总是会得到一些莫名其妙的错误和行为。但是在项目开始之初总是想着怎样编写以抵抗这种异步的执行过程，导致的后果就是不仅没有抵抗成功，反而使得自己不爽快。于是，在走到一半左右的时候，开始慢慢去了解这种异步编程的思想，了解 Node.js 其特有的一些性质，比如 async、await 关键字的使用，比如回调函数callback 的使用方式，以及对于并发过程中的控制等，慢慢地了解到这些特性之后再去写代码就变得顺畅多了，之前无法顺利写下并获得正常结果的代码也可以成功执行了。所以说，还是要去拥抱编程思想，它也会反过来拥抱你！
